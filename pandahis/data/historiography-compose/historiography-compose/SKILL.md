# 史略聚合撰写师 v3

## 定位

Phase 3 = 三阶段流水线的**最终聚合器**。

```
Phase 1 索引 JSON ───→ 名称/分类/优先级/关联
Phase 2 召回原文 ───→ 史料原文（内部带段落号）
Phase 3 聚合 ────────→ 时空补全 + 详情 + 最终记录
        ↓
   一条完整 Bitable 记录
```

**输入**：Phase 1 产出的 `索引/[书名]_[卷号].json` + Phase 2 召回的原文
**输出**：`产出/[书名]_[卷号]_史略记录.json`（每条记录一个完整 Bitable 行）

---

## 一、最终产出格式（🔥 唯一输出标准）

每条记录 = Bitable 史略表的一行，字段必须全部填充，无空值。

```json
{
  "records": [
    {
      "史略ID": "HXWD0001",
      "史略名称": "黄帝",
      "史略简介_20字以内": "华夏始祖，败炎帝灭蚩尤，被尊天子",
      "史略分类": "君纪",
      "一级文明归属": "华夏",
      "二级王朝归属": "五帝",
      "三级帝王归属": "黄帝",
      "史略开始年": -2698,
      "史略结束年": -2598,
      "时间精度": "B",
      "时间备注": "推算自黄帝作为五帝之首的传说年代",
      "优先级": "P0",
      "优先级判定理由": "华夏文明开创者，拐点级人物",
      "主要史料出处": "史记·卷001",
      "史料原文": "黄帝者，少典之子，姓公孙，名曰轩辕。生而神灵……（Phase 2召回原文，内部保留段落号标注）",
      "详情": "黄帝这个人，在咱们华夏历史上，算得上开天辟地头一号人物……（《明朝那些事儿》风格翻译全文）",
      "审核摘要": "确定",
      "更新时间": "2026-05-28T08:42:00+08:00",
      "v4_flash": "",
      "ds_v4_pro": ""
    }
  ]
}
```

### 字段来源映射

| 字段 | 来源 | 生成方式 |
|------|------|---------|
| 史略ID | Phase 3 生成 | HX{朝代缩写}{自增序号}，规则见二 |
| 史略名称 | Phase 1 | 直接复制 entity |
| 史略简介_20字以内 | Phase 1 | 将 bio 压缩至 20 字以内 |
| 史略分类 | Phase 1 | 直接复制 type |
| 一级文明归属 | Phase 3 补全 | 据 entity 上下文判断（史记→华夏），查 reference 文件 |
| 二级王朝归属 | Phase 3 补全 | 据 start_year 匹配 reference 朝代区间 |
| 三级帝王归属 | Phase 3 补全 | 据 start_year 匹配 reference 帝王在位区间 |
| 史略开始年 | Phase 3 补全 | 从原文提取/推算，查 reference 帝王在位区间 |
| 史略结束年 | Phase 3 补全 | 从原文提取/推算，查 reference 帝王在位区间 |
| 时间精度 | Phase 3 补全 | A/B/C/D 四级降级，规则见三 |
| 时间备注 | Phase 3 补全 | 一句话说明时间来源 |
| 优先级 | Phase 1 | 直接复制 priority |
| 优先级判定理由 | Phase 1 | 直接复制 priority_reason |
| 主要史料出处 | Phase 3 | 格式：{著作}·{卷号}，如 史记·卷001 |
| 史料原文 | Phase 2 | 召回原文，内部保留 [Pn] 段落号标注 |
| 详情 | Phase 3 撰写 | 《明朝那些事儿》风格翻译整合 |
| 审核摘要 | Phase 1 | 直接复制 confidence |
| 更新时间 | Phase 3 | 当前时间戳（ISO 8601 + 时区） |
| v4_flash | Phase 3 | 暂留空字符串 |
| ds_v4_pro | Phase 3 | 暂留空字符串 |

### 🔥 绝不空值原则

所有字段必须有值。Phase 1 传递过来的字段如果为空，按以下降级：
- 年号类 → D 级（文明最宽区间）
- 文本类 → 最短兜底（如简介降为"待补"，帝王降为朝代）
- 布尔/枚举 → 默认值

## 二、史略ID 生成规则

### 格式：`{文明缩写}{朝代缩写}{4位自增序号}`

- 文明缩写：华夏 = HX（当前唯一，后续文明扩展时追加）
- 朝代缩写：两字母编码（见下表）
- 序号：该文明+朝代组合内，从 0001 起自增

### 朝代缩写映射表

```
WD 五帝      XC 夏        SC 商        XZ 西周      CQ 春秋
ZG 战国      QC 秦        XH 西汉      XN 新        DH 东汉
SG 三国      XJ 西晋      SL 十六国    DJ 东晋      NB 南北朝
SU 隋        TC 唐        WJ 五代十国  BS 北宋      LC 辽
XX 西夏      JC 金        NS 南宋      DL 大理      BH 渤海
NZ 南诏      YC 元        MC 明        HJ 后金      QN 清
NM 南明      MG 民国      XG 新中国
```

### 生成步骤

1. 从 Phase 1 记录中读取 `civilization` 和 `dynasty`
2. 查表获取文明缩写 + 朝代缩写
3. 在本批次产出文件中，扫描已生成的同朝代 ID，取最大值 +1
4. 拼接为完整 ID，如 `HXWD0001`

## 三、时空字段补全规则（🔥 Phase 3 独有）

### 3.1 运行前准备

执行前必须读取 `reference/文明朝代帝王.json`，获取：
- 文明列表及选项
- 各朝代的时间区间（start_year / end_year）
- 各帝王的在位区间（start_year / end_year）

### 3.2 时间精度降级链（绝不空值）

| 等级 | 来源 | 示例 | 可靠性 |
|-----|------|------|-------|
| A | 原文明确纪年 | "汉高祖五年"→前202 | 高 |
| B | 帝王在位推算 | "尧时"→尧在位区间 | 中 |
| C | 朝代区间估计 | "夏时"→夏朝起止 | 低 |
| D | 上下文最宽推断 | 连朝代都无法确认 | 极低 |

🔥 **start_year 和 end_year 绝不能为空。** 按以下链条逐级降级：

1. **A 级**：原文有明确纪年 → 直接填入
2. **B 级**：原文提及帝王名 → 用帝王在位区间（从 reference 文件查）
3. **C 级**：只能确认朝代 → 用朝代起止年份
4. **D 级**：连朝代都无法确认 → 用所属文明的最宽区间（如华夏 -3000~2100）

### 3.3 匹配流程

对每条史略记录：

1. 确认 `一级文明归属` — 史记所有内容→华夏文明
2. 据 Phase 1 索引中的 entity 及原文上下文，判断该史略所属的帝王
3. 在 reference 文件中查找帝王 → 获取 start_year / end_year
4. 帝王所在的朝代 → 获取 dynasty 字段
5. 人物：start = 最早活跃时间，end = 最晚活跃时间（无卒年用帝王在位结束年）
6. 事件：start = end = 事件发生年份（无法精确到年则用帝王区间）
7. 典制/民录/著作：用记载所在帝王在位区间

### 3.4 补全后字段

每条记录补全输出：
```json
{
  "一级文明归属": "华夏",
  "二级王朝归属": "五帝",
  "三级帝王归属": "黄帝",
  "史略开始年": -2698,
  "史略结束年": -2598,
  "时间精度": "B",
  "时间备注": "推算自黄帝作为五帝之首的传说年代"
}
```

## 四、详情质量底线（🔥 强制执行，不达标驳回重写）

### 4.1 最低字数门槛

```
详情最低字数 = 关联段落数 × 150 字
```

| 关联段落数 | 最低字数 | 典型示例 |
|-----------|---------|---------|
| 1-2 段 | 300 字 | 轻量事件/制度 |
| 3-5 段 | 750 字 | 士臣人物 |
| 6-15 段 | 1500 字 | 一般君纪 |
| 16+ 段 | 2500+ 字 | 舜（20段，应达3000字+） |

🔥 **不达标 = 驳回重写，直至达标。**

### 4.2 逐段覆盖验证

原文每一段的核心信息，必须在详情中找到对应句。写完后逐段核对：

```
[P1] 核心信息：黄帝出身、神农世衰、修德振兵、阪泉之战、三战得志
      → 详情是否覆盖？ □ 出身 □ 背景 □ 练兵 □ 战役 □ 结果
[P2] 核心信息：蚩尤作乱、徵师诸侯、禽杀蚩尤
      → 详情是否覆盖？ □ 作乱 □ 徵师 □ 擒杀
```

**任何一段的核心信息有遗漏 → 驳回补写。**

### 4.3 君纪人物首段要求

君纪人物详情首段必须包含三个要素：
1. **出身**：家族/父母/出生地
2. **时代背景**：当时天下局势
3. **历史定位**：一句话概括此人的历史意义

### 4.4 内容质量规则

#### 延伸规则

可以延伸解读，但**只能基于确凿史实**：
- ✅ 《史记》其他卷记载、同时代正史、学界公认史料
- ✅ 后世公认的历史评价（如"天下明德皆自虞帝始"→ 舜被推为中国德政传统起点）
- ✅ 同一事件在不同卷的互补信息
- ❌ 自己推断的历史影响（"这标志着中国进入了XX时代"）
- ❌ 没有出处的背景知识补充
- ❌ 强行升华灌鸡汤（"这就是中国人的精神底色"）

**延伸铁律**：不确定的地方必须用"一说""可能""推测"，不能写成事实。

#### 反注水规则 🔥

写完逐段自查：**这段砍掉会丢信息吗？**

- 同一件事换不同角度说两遍 → 删
- "换句话说""也就是说""这意味着"后跟的内容跟前面意思一样 → 删
- 过渡句堆砌（"不仅如此""更值得一提的是""值得注意的是"）→ 删
- 白描已经够有力时硬加形容词 → 删

**口诀**：宁可干净利落 2000 字，不要注水 5000 字。字数底线是兜底的网，不是追求的目标。

### 4.5 驳回检查清单

以下任一项不满足 → 驳回重写：

- [ ] 字数 ≥ 关联段落数 × 150
- [ ] 原文每段核心信息全覆盖
- [ ] 延伸内容有史实依据（不确定的有标注）
- [ ] 无重复表述、无注水凑字数
- [ ] 君纪首段含出身+背景+定位
- [ ] 初次出场关键人物有简要介绍
- [ ] 生僻字首次出现有注音
- [ ] 古地名首次出现标注今地
- [ ] 无心理活动编造
- [ ] 无原文没有的对话
- [ ] 无【】标记

## 五、详情撰写规则（翻译整合）

### A 层：翻译规则（完整继承）

Phase 3 撰写"详情"时必须遵守 `references/翻译规则_v3_完整版.md` 的全部规则。

| 规则 | 适用范围 | 备注 |
|------|------|------|
| 规则一·风格定位 | 完全适用 | 明朝那些事儿笔调 |
| 规则二·结构完整 | 完全适用 | 出身→事迹→去世 |
| 规则三·数据准确 | 完全适用 | 数字年代精确保留 |
| 规则四·原文引用规范 | 完全适用 | 自然融入，无【】标记 |
| 规则五·人物刻画 | 完全适用 | 不神化不矮化，指代多样化 |
| 规则六·纯叙事 | 完全适用 | 无小标题/分节号/列表 |
| 规则七·注音标注 | 完全适用 | 生僻字首次标拼音 |
| 规则八·六类触发 | 完全适用 | 制度/地名/人名/器物/事件/补充 |
| 规则九·通假字处理 | 完全适用 | 首次出现标注 |
| 规则十·逐句顺译 | **调整** | 见下方"规则十调整" |
| 规则十一·写作自检 | 完全适用 | 写完后逐条检查 |

### 规则十调整：主题内逐句顺译（🔥强制执行）

Phase 3 输入是 Phase 2 召回的多段散装原文，不是一整卷。

**调整规则**：

1. 将召回段落按**时间线/主题逻辑**排列
2. 每个主题内，覆盖所有召回段落的每一句信息
3. 多段重复的相同内容 → 取最完整版本，不重复写
4. 不因"这一段前面看过了"就跳过——除非是完全相同的重复叙述

#### 逐句覆盖清单（强制执行，不可跳过）

**步骤0：拆原文为逐句清单**
将 Phase 2 召回的所有段落拆成逐句清单并编号：
```
P13-01: 帝喾娶陈锋氏女，生放勋
P13-02: 娶娵訾氏女，生挚
...
P33-01: 於是尧乃试舜五典百官皆治
```
共N句。这个清单只用于内部覆盖追踪，不写入正文。

**步骤0.1：强制读取翻译规则**
打开 `references/翻译规则_v3_完整版.md`，从头读到尾。禁止凭记忆跳过。

**步骤1：逐句翻译**
对着逐句清单，按时间/逻辑顺序翻译。翻完一句在清单上打√。禁止跳句、漏句。

**步骤2：覆盖检查**
写完后逐句核对：清单上的编号都翻了吗？
- ✅ 全部√完 → 进入自检
- ❌ 有遗漏 → 回到正文补入，直到全部√完

### B 层：Phase 3 独有规则

#### B1. 多源整合

- 按时间线排列所有片段
- 同一事件的多个描述 → 主叙述用最详细的版本
- 不同卷对同一事件有矛盾记载 → 主叙述用最详细版，正文中说明分歧：
  「按《史记·XX》的说法…不过《史记·YY》的记载不太一样，说是…」
- 只在信息量有差异时才标注多源

#### B2. 反幻觉自检（强制执行）

输出后逐条自答这5个问题：

1. **有没有原文没有的心理活动？**
2. **有没有原文没有的对话？**
3. **有没有原文没有的时间/地点/数字？**
4. **有没有把推测写成事实？**
5. **有没有混入不同来源没标注？**

5个答案全是"没有"→ 通过。

#### B3. 出处可溯

写完后用原文段落号做覆盖率检查。列出 Phase 2 召回的所有段落号，逐一确认每段的核心信息有没有被吸收。

## 六、执行流程

```
Phase 1 索引 JSON ──→ 读入每条记录的元数据
Phase 2 召回原文 ──→ 读入原文段落
        ↓
步骤0：读取 reference/文明朝代帝王.json（时空匹配）
步骤1：读取翻译规则文件（强制执行）
步骤2：对每条 Phase 1 记录，逐条处理：
  a. 时空补全：据原文上下文查 reference，匹配 civilization/dynasty/emperor/start_year/end_year
  b. 时间精度标注：A/B/C/D 降级 + time_note
  c. 查朝代缩写表，生成史略ID
  d. 压缩简介至20字以内
  e. 组装主要史料出处（著作·卷）
  f. 放入史料原文（Phase 2召回内容）
  g. 撰写详情（规则十逐句覆盖 + 反幻觉自检）
  h. 填入更新时间戳
  i. 检查所有字段非空
步骤3：输出完整 JSON 到 产出/ 目录
```
