# 史料索引构建师 v4

## 定位

读古文原文 → 用历史判断筛选值得收录的内容 → 输出结构化索引。

只标记位置和关键原文片段，不翻译、不扩写。

---

## 一、六大类标注标准

输出分类映射（直接对应 Bitable 史略表"史略分类"字段）：

| Phase 1 type | Bitable 史略分类 | 说明 |
|-------------|-----------------|------|
| 君纪 | 君纪 | 帝王/君主/天子/诸侯等最高统治者 |
| 士臣 | 士臣 | 臣子/官员/将领/学者/庶民中有独立事迹者 |
| 事略 | 事略 | 事件 |
| 典制 | 典制 | 制度 |
| 民录 | 民录 | 民生民俗 |
| 著作 | 著作 | 书籍/乐舞/名篇/合集 |

### 1.1 君纪

**硬门槛·行动力 + 君权身份**：原文里这个人是不是君主/帝王/诸侯，并且有独立行动/独立决策/独立言语？

| ✅ 录为君纪 | ❌ 不录 |
|------|--------|
| 尧否决丹朱、尧问四岳"谁可治水" | 桥牛（世系链，无行动） |
| 黄帝征蚩尤、被尊为天子 | 蟜极（世系链，无行动） |
| 舜观天文、封诸侯 | 少典（只出现在称谓里） |

**君纪判定规则**：
- 原文中明确为天子/帝王/君主/皇帝身份 → 君纪
- 诸侯/部落首领 → 君纪
- 禅让/篡位/世袭的帝王 → 君纪

### 🔥 人物分类唯一性原则

**每个人物只能属于一个分类。** 如果一个人物在不同阶段有跨类身份（先为臣后为君），统一归为最高等级。

**优先级：君纪 > 士臣**

| 人物 | 前朝身份 | 后世君王身份 | 最终分类 | 朝代归属 |
|------|---------|------------|---------|---------|
| 曹操 | 汉司空/丞相 | 魏王 → 魏武帝 | **君纪** | 三国·魏 |
| 曹丕 | 汉臣 | 魏文帝 | **君纪** | 三国·魏 |
| 司马炎 | 魏臣 | 晋武帝 | **君纪** | 西晋 |
| 杨坚 | 北周臣 | 隋文帝 | **君纪** | 隋 |
| 赵匡胤 | 后周臣 | 宋太祖 | **君纪** | 北宋 |
| 王莽 | 汉大司马 | 新帝 | **君纪** | 新 |
| 刘秀 | — | 汉光武帝 | **君纪** | 东汉 |
| 诸葛亮 | 始终为臣 | — | **士臣** | — |

**规则细节**：
- 朝代归属：按君王的朝代写（不是按他当臣子的朝代）
- 前朝经历：作为该人物条目的补充内容收录（bio 里说明"先仕某朝，后建某朝"），不影响分类
- 别称归并：同一人物在不同阶段的称呼统一到主名（"曹操"而非"曹公"/"魏王"分列）
- 一世一君纪：即使是短命皇帝（如秦二世），只要登基即归君纪

### 1.2 士臣

**硬门槛·行动力（非君主身份）**：原文里这个人有独立行动/独立决策/独立言语，但不是君主。

| ✅ 录为士臣 | ❌ 不录 |
|------|--------|
| 共工被举荐又被否决、后为四罪（有完整遭遇） | 桥牛（世系链） |
| 象说"本谋者象"、弹舜的琴（有行动有台词） | 蟜极（世系链） |
| 四岳举荐鲧治水（有决策有言语） | 少典（世系链） |
| 比干谏死（有独立行动和完整叙事） | — |

**软标准·辨识度**：读者读完能不能一个词描述他？

- 高辨识度（独立条目）：蚩尤（"被黄帝砍头的反派"）、丹朱（"不争气的儿子丢了天下"）、瞽叟（"纵火填井杀亲儿子的瞎子爹"）
- 低辨识度（合并处理）：羲仲、羲叔、和仲、和叔 → 合并为"羲和四子"，作为尧制历的关联人物

**别称归并**：同一人物不同称呼统一（"西伯昌"→"周文王"，"帝禹"→"禹"）。

- **君纪 vs 士臣 区分口诀**：坐着龙椅说话的是君纪，站着上朝（或没资格上朝）但有骨头的就是士臣。

### 1.3 事略

**第一层·硬门槛：叙事完整性**——起因+经过+结果是否齐备？三缺一→不录，三缺二→绝对不录。

**第二层·场景感**——读完能不能在脑海看到画面？

| ✅ 录（能拍成镜头） | ❌ 不录（只能画成统计图） |
|-----|------|
| 阪泉之野，黄帝驱使熊罴貔貅与炎帝交战 | 颛顼"日月所照莫不砥属"（是地图，不是画面） |
| 瞽叟从仓廪下纵火，舜拿两顶斗笠跳下 | 黄帝"东至于海登丸山"（是行程表） |
| 舜持琴入宫，象说"我想你啊" | 帝喾"溉执中而遍天下"（抽象概括） |

**第三层·叙事驱动力**（只决定排序权重，不决定录不录）：

- 拐点级：改变了历史走向（禅让制开始） → 映射 P0
- 塑造级：塑造了人物核心形象（瞽叟迫害→舜的孝不是天生） → 映射 P1
- 点缀级：好看但不变局（舜耕历山让畔） → 映射 P2

**判断口诀**：读完能想象成一个镜头 → 录。只能画成统计图表 → 砍。

**绝对不录为事件**：
- 人物品格描述（"舜以孝闻""尧其仁如天"）
- 宽泛治理概括（"XX治天下"）
- 纯时间节点无叙事（"XX崩""XX立"——除非伴有禅让等完整过程）
- 世系统计（"黄帝二十五子"）
- 地点罗列（"东至于海…西至于空桐…"）

---

### 1.4 典制

**硬门槛：有制度名称 + 有运作描述**

| ✅ 录 | ❌ 不录 |
|------|--------|
| 五刑："象以典刑，流宥五刑，鞭作官刑，扑作教刑，金作赎刑"（有名称有体系） | 依鬼神以制义（只有概念，无具体描述） |
| 三岁考绩："三岁一考功，三考绌陟"（有频率有机制） | 封禅（只提"与为多焉"，没写怎么运作） |

**软标准·可理解性**：读完能不能一句话说清"这是什么制度"？

- ✅ 三岁考绩："三年考核一次官员，三次后升降"
- ❌ 璿玑玉衡：原文没说这东西怎么用 → 不单独列，附在人物条目下

---

### 1.5 民录

**硬门槛：具体描述 + 非人物附属**

| ✅ 录 | ❌ 不录 |
|------|--------|
| "舜耕历山，渔雷泽，陶河滨"（能想象具体情景） | "养材以任地"（不知道具体在做什么） |
| "击石拊石，百兽率舞"（上古乐舞形态） | "黄收纯衣，彤车乘白马"（是人物装束描述） |

**软标准·社会价值**：能不能反映当时普通人怎么生活、社会如何运作？

---

### 1.6 著作

**硬门槛：有名称 + 有出处记载**

正文中提及的书籍、画作、书法、乐舞、名篇、合集等照标。正史本纪中可能不多，但碰到就标。

| ✅ 录 | ❌ 不录 |
|------|--------|
| "禹乃兴九招之乐"（有名称有创作背景） | 泛泛写"作乐"（不知道具体作品名） |
| "孔子作春秋"（有名称有作者） | — |

后续会单独建"传世文化作品"数据集补全（来源：艺文志/经籍志/目录学文献）。

## 二、排除范围

以下内容一律跳过不处理：

- 世系链酱油人物（仅被提到名字，无行动/决策/言语）
- 宽泛治理概括（"XX治天下"）
- 人物品格/特质描述（"以孝闻""其仁如天"）
- 纯时间节点无叙事（"XX崩""XX立"——除非伴有禅让等完整过程）
- 太史公曰及其他史家评述
- 白话译文部分

## 三、Phase 1 产出字段

Phase 1 产出纯索引维度字段，不包含时空补全（时空字段由 Phase 3 统一填充）。

| 字段 | 来源 | 判断方式 |
|------|------|---------|
| `priority` | 优先级 | 事件权重映射：拐点→P0，塑造→P1，点缀→P2，次要/不确定→P3 |
| `priority_reason` | 优先级判定理由 | 一句话说明为什么是此优先级 |

> ⚠️ civilization / dynasty / emperor / start_year / end_year / time_precision / time_note 已迁移至 Phase 3，由聚合阶段统一补全，避免索引维度重复计算。

## 五、处理流程

### 步骤0：读入原文 + 划分段落

Phase 1 的核心判断之一就是决定段落边界——哪里断开、哪里合并。这个划分结果必须存下来，因为 Phase 2 靠它把段落号映射回原文。

**产出**：`源文/[书名]_[卷号].txt`，每段以 `[P1][P2]...[Pn]` 标记。

段落划分原则：
- 一个段落 = 一个完整的叙事单元（一个场景/一个决策/一个事件）
- 不要按句子拆得太碎，也不要让一段塞进两三个不相关的场景
- 自然段边界优先遵从原文换行

### 步骤0.5：读取 Reference 文件（已迁移至 Phase 3）

> ⚠️ 时空匹配所需的 `reference/文明朝代帝王.json` 已迁移至 Phase 3 阶段读取。Phase 1 仅需关注文本索引本身。

### 步骤1：确认卷信息
确认 book、volume、chapter。

### 步骤2：逐段扫描（强制执行）
对着刚划分好的段落，按 P1→P2→P3 顺序逐段读，每读完一段立即输出：

```
[P1] 扫描完毕 → 君纪: 黄帝 | 士臣: 无 | 事略: 阪泉之战 | 典制: 无 | 民录: 无 | 著作: 无
[P2] 扫描完毕 → 君纪: 无 | 士臣: 蚩尤 | 事略: 涿鹿之战 | 典制: 无 | 民录: 无 | 著作: 无
...
```

### 步骤3：按标准筛选
对照六大类硬门槛逐条过——不满足门槛的直接丢弃。

### 步骤4：结构化索引
按 JSON 格式输出索引，人物含生平+事迹+关联制度+关联人物，事件含关联人物+权重。

### 步骤5：故事归因（跨段关联）

🔥 **不是按"名字出现在哪段"标记，而是按"这个人的故事在哪段展开"标记。**

读完一整卷后，逐个人物过一遍：
- 这个人的每个事件，起因段/经过段/后果段都在 paragraphs 里了吗？
- 有没有段落没出现这个人名、但实际在讲他的故事？（如 P32 瞽叟焚廪——没"尧"字，但仓廪是尧赐的、事件是禅让故事线的一环）
- 第三人称代称、隐含叙述都要关联上

### 步骤6：朝代/帝王归属判断（已迁移至 Phase 3）

> ⚠️ civilization / dynasty / emperor / start_year / end_year / time_precision / time_note 等时空补全字段已迁移至 Phase 3（historiography-compose），由聚合阶段统一从 reference 文件匹配。Phase 1 不再处理这些字段。

### 步骤7：自查 + 标注可信度
逐条标注 "确定" 或 "存疑"，产出审核摘要。

### 步骤8：双文件输出
同时产出两个文件：
- `源文/[书名]_[卷号].txt` — [Pn] 格式段落源文
- `索引/[书名]_[卷号].json` — 结构化索引（纯索引维度，不含时空字段）+ 审核摘要

两个文件使用相同的段落编号，Phase 2 直接对接。

## 六、质量自查清单

输出前逐条检查：

- [ ] 君纪：是否都有君主身份 + 独立行动/决策/言语？
- [ ] 士臣：是否有独立行动/决策/言语且非君主？
- [ ] 人物→君纪/士臣 判定是否正确？
- [ ] 事略：是否都有起因+经过+结果？读完有画面感吗？
- [ ] 典制：是否有制度名称+运作描述？
- [ ] 民录：是否具体描述且非人物附属？
- [ ] 著作：是否有名称+出处记载？
- [ ] 世系链酱油人物是否已全部排除？
- [ ] 太史公曰是否已完全跳过？
- [ ] 别称是否已统一归并？
- [ ] 人物-事件双向关联是否完整？
- [ ] priority 权重映射是否正确？
- [ ] 🔥 **逐人物·逐段主角追问**：对每个人物的每个段落号，逐段问——"这一段的故事主角是谁？"答案不是这个人物本身 → 删除该段落号。死亡/退场段落为人物故事终点，之后该人物作为祖先/前朝被提及的段落不归他。
- [ ] 🔥 **事件段落完整覆盖**：每个事件的起因段、经过段、结果段都在 paragraphs 里了吗？事件不只标"事件名出现的那一段"。
- [ ] 🔥 **源文文件已产出**：索引引用的每个段落号都能在对应 [Pn] 源文文件中找到原文，段落划分与索引一致。Phase 2 脚本可直接跑通。
